Fail2ban Documentation

What is Fail2ban?

Fail2ban scans log files and bans IPs that show malicious signs (too many password failures, seeking exploits, etc.). It updates firewall rules to reject the IP addresses for a specified time.

Common Jails:

  • sshd - SSH brute force protection
  • nginx-http-auth - HTTP basic authentication failures
  • nginx-botsearch - Block bots searching for vulnerabilities
  • django - Django login failures

Configuration Files:

  • /etc/fail2ban/jail.conf - Default configuration (don't edit)
  • /etc/fail2ban/jail.local - Your custom configuration
  • /etc/fail2ban/filter.d/ - Log parsing filters

Useful Commands:

# Check fail2ban status
sudo fail2ban-client status

# Check specific jail
sudo fail2ban-client status sshd

# Unban an IP
sudo fail2ban-client set sshd unbanip 192.168.1.100

# Reload fail2ban
sudo systemctl reload fail2ban

Manual Update from CLI
Recommended: Use the Update Script

The easiest way to update is using our automated update script. It checks everything first, then runs the update safely:

cd /path/to/your/huduglue
# (use find command below if you don't know the path)
./update.sh

The script will:

  • ✓ Check you're in the correct directory
  • ✓ Verify git repository and virtual environment
  • ✓ Warn about uncommitted changes
  • ✓ Pull updates from GitHub
  • ✓ Install dependencies
  • ✓ Run migrations
  • ✓ Collect static files
  • ✓ Restart the service
  • ✓ Show current version
Show Manual Step-by-Step Instructions

For advanced users who prefer manual updates via command line:

Important: Copy and run these commands ONE AT A TIME, not all at once. Wait for each command to complete before running the next one.
Step 1: Navigate to Application Directory

Find where your application is installed:

find /home -name "manage.py" -type f 2>/dev/null | grep huduglue
# This will show the path to your installation

Then navigate to that directory (replace with YOUR path from above):

cd /path/to/your/huduglue
# Example: cd /home/administrator/huduglue
# OR: cd /home/administrator
Step 2: Verify Location
ls manage.py venv/
# Both should exist. If not, you're in the wrong directory!
Step 3: Pull Updates
git pull origin main
Step 4: Apply Updates
source venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py collectstatic --noinput
sudo systemctl restart huduglue-gunicorn.service
Important Notes:
  • Find your path first! The application path varies by installation. Use the find command in Step 1 to locate YOUR installation.
  • Always verify with ls manage.py venv/ that you're in the correct directory before proceeding
  • Run commands ONE AT A TIME - don't paste them all at once
  • Check for uncommitted changes: git status before pulling updates
  • If you get "externally-managed-environment" error, make sure you activated the venv first
